## 二、与 Ant Messenger 对比的差异清单

### 🔴 P0 — 必须有（核心功能缺失）

| # | 缺失功能 | 说明 | 影响端 |
|---|---------|------|--------|
| 1 | **好友请求/验证系统** | 无"添加好友"流程（搜索→发送请求→对方同意/拒绝），当前直接双向加好友，没有请求审批环节 | 全端 |
| 2 | **消息撤回** | 无法撤回已发送消息（只有删除，删除无撤回提示） | 全端 |
| 3 | **已读/未读状态** | 无消息已读回执（privacy_read_receipt字段已存在但前端未实现），用户不知道消息是否被对方看到 | 全端 |
| 4 | **消息引用/回复** | 无法引用某条消息回复，群聊中无法定位回复对象 | 全端 |
| 5 | **群聊管理增强** | 无群公告、@所有人、群名片、群禁言、群审批加入 | 全端 |
| 6 | **图片/视频预览大图** | 当前只展示缩略图，无点击放大/滑动浏览/保存到相册 | 全端 |
| 7 | **消息通知推送** | 无本地推送（Foreground/Background通知），Web端无Browser Notification，锁屏无新消息提醒 | 全端 |
| 8 | **跨平台一致性** | Android/iOS/Web/Windows UI差异大，未做统一响应式布局 | 全端 |

### 🟡 P1 — 应该有（增强体验功能）

| # | 缺失功能 | 说明 | 影响端 |
|---|---------|------|--------|
| 9 | **朋友圈/动态流** | 无类似微信朋友圈功能（发图文动态、朋友点赞评论） | 全端 |
| 10 | **红包/转账** | 无聊天内红包发送、转账功能 | 全端 |
| 11 | **实时位置共享** | 无发送位置、实时位置共享功能 | 移动端 |
| 12 | **小程序/内嵌WebView** | 无小程序框架或内嵌H5能力 | 全端 |
| 13 | **公众号/订阅号** | 无公众号关注、消息推送系统 | 全端 |
| 14 | **聊天背景个性化** | 聊天背景设置API存在但前端未实现UI | 全端 |
| 15 | **消息多选批量操作** | 批量选择消息→转发/删除/收藏（前端已有选择模式但功能不完整） | 全端 |
| 16 | **群文件/共享文件** | 群聊无共享文件管理面板 | 全端 |
| 17 | **群投票** | 无群聊投票功能 | 全端 |
| 18 | **消息快捷回复/表情回应** | 长按消息→选择表情回应 | 全端 |
| 19 | **语音通话（非视频）** | 只有视频通话，无纯语音通话模式 | 全端 |
| 20 | **群视频通话/多人通话** | 只支持一对一，无群组音视频通话 | 全端 |
| 21 | **通讯录同步** | 无手机通讯录→好友匹配建议 | 移动端 |
| 22 | **二维码/扫码** | 无扫一扫、我的二维码加好友 | 移动端 |
| 23 | **多语言/国际化** | 只有中文硬编码，无i18n支持 | 全端 |
| 24 | **深色模式完善** | 已有暗主题切换但很多组件未适配 | 全端 |
| 25 | **消息加密** | 无端到端加密（E2EE），消息明文传输 | 全端 |
| 26 | **聊天记录导出/备份** | 无导出聊天记录、云端备份 | 全端 |
| 27 | **桌面端独立布局** | Windows和Web端与移动端共用UI，无桌面优化：多窗口、拖拽文件发送、快捷键 | 桌面端 |
| 28 | **Web端PWA离线支持** | 虽然有manifest.json但无ServiceWorker离线缓存 | Web |
| 29 | **Web端断网检测** | 无显式网络状态提示/重连状态UI | Web |

## 六、推荐开发顺序

### 第一阶段（P0快速补齐）— 预计2周
```
1. 好友请求/验证系统（搜索→请求→同意→加好友）
2. 已读回执显示（利用现有privacy_read_receipt设置）
3. 消息撤回（within 2 minutes）
4. 消息引用/回复
5. 图片/视频预览大图
6. 本地通知推送
7. iOS Push Notification（APNs）
8. iOS ATS/Safe Area适配
```

### 第二阶段（P1核心增强）— 预计3周
```
9. 群管理增强（公告、@、禁言、审批加入）
10. 朋友圈/动态（发图文+点赞评论）
11. 红包/转账功能
12. 语音通话模式（纯音频）
13. 聊天背景个性化UI
14. 深色模式全面适配
15. 二维码/扫码加好友
16. 桌面端独立布局（Web/Windows）
17. 多语言/国际化（i18n）
```

### 第三阶段（P2完善）— 预计2周
```
18. 消息多选批量操作完善
19. 视频消息
20. 群投票/群文件
21. 表情回应（Reaction）
22. 拖拽发送文件（Web/桌面）
23. Windows系统托盘
24. 键盘快捷键
25. 聊天记录导出/备份
26. PWA ServiceWorker
27. Face ID/Touch ID应用锁
28. CallKit
```

### 持续优化
- 代码重构（模块化拆分大型页面，如chat_page.dart 1300行）
- 性能优化（消息列表虚拟化、图片懒加载、内存管理）
- 自动化测试补全
- CI/CD部署流程
- 单元测试和Widget测试

## 七、当前代码质量问题（顺便指出的）

| # | 问题 | 文件 | 建议 |
|---|------|------|------|
| 1 | chat_page.dart 1300行 | `lib/pages/chat_page.dart` | 应拆分为 widget/mixin/controller |
| 2 | ApiService单例全局状态 | `lib/services/api_service.dart` | 考虑 Provider/Riverpod状态管理 |
| 3 | 硬编码服务器URL | `lib/services/api_service.dart:15` | 改为可通过后台切换 |
| 4 | 无统一错误处理 | 多处 | 添加全局错误拦截器 |
| 5 | 无路由管理 | `main.dart` | 引入GoRouter或Navigator 2.0 |
| 6 | 无状态管理方案 | 全项目 | 引入Riverpod/Bloc替代直接setState |
| 7 | 无API版本控制 | `server/index.js` | API路由加版本号 /api/v1/ |
| 8 | SQLite无连接池 | `server/index.js` | 对高并发场景需考虑连接池或迁移到PostgreSQL |
